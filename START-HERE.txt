FACELESS STUDIO
===============

Full manual is in README.md. This is the short version.

Everything is one command:   faceless <verb>

    faceless start      open the control panel
    faceless check      make sure this machine is set up right
    faceless --help     the full list

That command is installed by the setup script. If it is ever missing,
"python make_video.py <step>" always works instead (python3 on macOS).


===============================================================
FIRST TIME - WINDOWS  (about 20 minutes)
===============================================================

1. Install Python 3.12
   https://www.python.org/downloads/

   On the FIRST installer screen, tick "Add python.exe to PATH".
   Easy to miss, and nothing works without it.

2. Install ffmpeg. In PowerShell:

       winget install Gyan.FFmpeg

   Close PowerShell, open it again, then check:

       ffmpeg -version

   Use the "full" build. The "essentials" build has no text support and
   burned-in captions will not work.

3. Install the Python side. In the project folder:

       .\setup.bat

   Several GB - leave it running. It installs Chatterbox first, then
   replaces its CPU-only PyTorch with the CUDA build for your graphics
   card, then proves the GPU works by running a real calculation on it.

   You need the NVIDIA driver. You do NOT need the CUDA Toolkit.

4. Add your free API keys. Copy config.example.json to config.json and
   paste them in:

       Pexels    https://www.pexels.com/api/         (required)
       Pixabay   https://pixabay.com/api/docs/       (required)
       Gemini    https://aistudio.google.com/apikey  (optional)

5. Check it:

       faceless check


===============================================================
FIRST TIME - macOS  (about 15 minutes)
===============================================================

Open Terminal (Cmd+Space, type "Terminal"):

    cd ~/Documents/faceless-pipeline
    brew install ffmpeg
    bash setup.sh
    cp config.example.json config.json
    open -e config.json

Paste your keys as in step 4 above, save, then:

    faceless check


===============================================================
WHAT DOCTOR SHOULD SAY
===============================================================

Everything should read "ok". The two lines that matter most:

  ok chatterbox  installed - NVIDIA GeForce RTX 5060 Ti - 17.1 GB VRAM
  ok captions    ass: libass present

If chatterbox says "CPU" but you have an NVIDIA card, PyTorch lost its
CUDA build. This is the most common problem and it is silent - nothing
errors, voicing is just 20x slower. Fix:

    python -m pip install --force-reinstall torch torchaudio --index-url https://download.pytorch.org/whl/cu128

Ignore the dependency-conflict warning it prints. Chatterbox pins an old
torch that cannot drive a modern card; overriding it is deliberate.

Run doctor again after ANY pip install in this project. It is the only
thing that catches this.


===============================================================
THE VOICE  (do this once)
===============================================================

Narration is cloned from a reference clip. Three are included. To add
your own, drop it in voices_refs/ - it must be audio you hold rights to:

    your own voice (30 seconds on a phone is plenty)
    Mozilla Common Voice   https://commonvoice.mozilla.org/   (CC0)
    LibriVox               https://librivox.org/              (public domain)

Open the control panel, go to the Voices panel, preview each one, and
choose one per language. Nothing will voice until you do.

Check your machine's speed before committing to a full video:

    faceless bench --lang en

Watch the it/s figure on the five sampling lines. Steady numbers are
healthy. The first line is always slower - that is GPU warm-up and is
reported separately.


===============================================================
EVERY DAY
===============================================================

Open the control panel:

    faceless start

The panel has four sections in the left sidebar:

    Dashboard   every video, and how far along each language is
    Activity    live log and exact numbers while something is running
    Voices      audition and choose a reference clip per language
    Settings    machine check, GPU, API keys, finished files

A normal day:

    0. Dashboard -> + New from script          (optional)
    1. Open the project -> Find visuals
    2. Review & swap any picture that doesn't fit
    3. Build every language - or just re-render one

There is a light/dark toggle at the bottom of the sidebar.

Leave the terminal open while you work. Ctrl+C when finished.

Double-clicking Start.bat (Windows) or Start.command (macOS) does exactly
the same thing, if you prefer that.


===============================================================
OR DO IT ALL FROM THE COMMAND LINE
===============================================================

    faceless start                     open the control panel
    faceless check                     check the machine
    faceless sheets                    show your projects
    faceless bench  --lang en          time the voice engine
    faceless voices --lang en          list/choose reference clips

    faceless new     --script my_script.txt --id video06
    faceless visuals --sheet sheets\\video06_MASTER_production_sheet.md
    faceless voice   --sheet sheets\\video06_MASTER_production_sheet.md --lang en
    faceless render  --sheet sheets\\video06_MASTER_production_sheet.md --lang en --captions

    faceless all     --sheet sheets\\video06_MASTER_production_sheet.md --lang en --yes

(macOS: use forward slashes in paths. The command itself is identical.)

Every verb has a long form too - "faceless start" is exactly
"python make_video.py studio". Use whichever you like.

Everything is cached. Re-running is cheap and picks up where it stopped.
Add --verbose to any command to see full errors.


===============================================================
IF SOMETHING BREAKS
===============================================================

Run this first - it usually names the problem exactly:

    faceless check

  "'python' is not recognized"                          (Windows)
      Not on your PATH. Re-run the Python installer, choose Modify,
      tick "Add python.exe to PATH".

  doctor says chatterbox is on CPU but you have an NVIDIA card
      See "WHAT DOCTOR SHOULD SAY" above. This is the common one.

  UnicodeEncodeError / 'charmap' codec can't encode          (Windows)
      Old console codepage. Should be handled automatically now; if you
      still see it, run  chcp 65001  first.

  "externally-managed-environment" when using pip             (macOS)
      Expected. Use  bash setup.sh  instead of pip. Never use
      --break-system-packages; it can break Homebrew itself.

  The whole computer freezes while voicing                    (macOS)
      The Mac GPU shares system memory and ran out. Force the CPU:
      FACELESS_DEVICE=cpu python3 make_video.py voice --sheet ... --lang en
      A PC with an NVIDIA card is much better suited to this job.

  Captions are not burned into the video
      ffmpeg was built without text support. On Windows use the "full"
      build. Or just upload the .srt next to the MP4 - YouTube handles
      that well and it is better for search anyway.

  Port 8765 already in use
      The studio is already running in another window.
